Java 면접 질문 정리(1)

자바 면접 질문 정리


  • Java정의
  • 객체지향적 언어 특징
  • 매서드 오버라이딩, 오버로딩
  • 접근제어자
  • Final 키워드란
  • 인터페이스와 추상 클래스
  • Static 이란

1.Java란


자바는 객체지향적 프로그래밍(OOP) 입니다.
객체지향적 언어(OOP)란, 프로그래밍에 사용될 데이터의 상태와 행위를 객체로 만들어, 객채간의 상호작용을 통해 비니지스 로직을 구성하는 프로그래밍 입니다.

2.객체지향적 언어 특징


  • 1.추상화


    객체의 공통적인 속성과 기능을 추출하여 정의하는것
    자바에서 추상화를 구현할 수 있는 문법 요소로는 추상 클래스(abstract) 와 인터페이가(interface) 있다

  • 2.상속


    상속이란 기존 클래스를 재활용하여 새로운 클래스를 작성하는 자바의 문법 요소
    클래스간 공유될수 있는 속성과 기능들을 상위 클래스로 추상화 시켜 해당 상위 클래스로부터 확장된 여러 개의 하위 클래스들이 모든 상위 클래스의 속성과 기능들 사용할 수 있도록 하는것
    반복적인 코드를 최소화하고 공유하는 속성과 기능에 간편하게 접근하여 사용할 수 있도록 함

  • 3.다향성


    객체의 속성이나 기능이 상황에 따라 여러가지 형테를 가질 수 있는것
    객체의 속성이나 기능이 그맥락에 따라 다른 역활을 수행할 수 있는 객체 지향의 특징
    그 예로는 메서드 오버라이딩(method overriding) 과 매서드 오버로딩(method overloading)이 있다.

  • 4.캡슐화


    서로 연관있는 속성과 기능들을 하나의 캡슐로 만들어 데이터를 외부로 부터 보호 하는것
    데이터 보호 - 외부로 부터 클래스에 정의된 속성과 기능들을 보호
    데이터 은닉 - 내부의 동작을 감추고 외부에는 필요한 부분만 노출

    캡슐화를 구현하기 위한 방법은 크게 2가지
    • 1.접근제어자
    • 2.getter/setter 메서드

3.매서드 오버라이딩, 오버로딩


매세드 오버라이딩(method overriding)


  • 부모클래스의 매서드를 하위클래스에서 재정의 하는것
  • 필요한 부분만 재 수정하여 원하는 기능으로 구현하는 것
  • 오버라이딩의 장점으로는 코드 중복을 줄이고, 기능 확장에 이점이 있음

매세드 오버로딩(method overloading)


  • 같은 이름을 가지는 매소드를 여러번 정의하는것
  • 매개변수의 개수나 타입을 다르게해 다른기능 으로 재정의 할 수 있음
  • 가동성이 좋아짐

4. 접근제어자


접근제어자는 클래스, 인터페이스, 맴버변수, 함수 등의 접근을 제어 합니다

외부 객체의 무분별한 접근으로 부터 내부 데이터를 보호할 수 있습니다

접근제어자 같은 클래스 맴버 같은 패키지 맴버 자식 클래스 그 외
Public O O O O
Protected O O O X
Default O O X X
Private O X X X


5.final 키워드란


변수나 메소드 또는 클래스가 변경불가능 하도록 만드는 자바 키워드

final을 사용하는 이유


클래스, 매서드, 변수가 변하지 않도록 하기 위해 사용

final을 선언하면 절대 변하지 않는 값 상수가 됩니다


6.인테페이스와 추상 클래스


추상 클래스(abstract)


추상 메서드 하나 이상을 포함한 클래스

  • 클래스이기 때문에 하나만 상속이 가능 합니다
  • 추상 메서드란, 선언부만 있고 구현부가 없는 메세드이며, 하위 클래스에게 메서드 구현을 강제함
  • 추상 클래스 -> extends 키워드 사용, 하나만 가능

인터페이스(interface)


구현체 없이 메서드에 대한 명세만 가지고 있고 모든 메서드 추상 메서드 입니다

  • 인터페이스를 상속받는 자식클래스에 모든 매서드에 대한 구현을 강제하며, 다중 상속이 가능합니다
  • 인터페이스 -> implements 키워드 사용, 여러개 가능


7.Static이란


static 변수와 static 메서드, 정적 맴버(=클래스 맴버)를 만들 수 있는 키워드 입니다

메모리에 한번 할당되어 프로그램이 종료될 때 해제되는것 입니다

일반적으로 만든 Class는 Static 영역에 생성되고, new 연산을 통해 생성된 객체는 Heap영역에 생성됩니다

가비지 컬랙터를 통해 Heap영역의 메모리는 관리 되지만 Static 영역에 할당된 메모리는 모든 객체가 공유 되는 장점을 가지지만 가비지 컬랙터를 통해 관리되지 않아 프로그램이 종료될깨 까지 존재 합니다
